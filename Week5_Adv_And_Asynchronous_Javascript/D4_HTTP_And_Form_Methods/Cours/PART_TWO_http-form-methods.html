Environnement HTTP : Explorer le Monde du Web

Imagine que ton navigateur est un explorateur qui utilise des routes (URLs) pour trouver des ressources sur des serveurs web. HTTP est le langage utilisé pour demander des ressources et recevoir des réponses du serveur.

Statuts HTTP : Les Panneaux de Signalisation
  • 200 OK ✅ : La demande a réussi. Exemple : `google.com` renvoie un 200 OK.
  • 404 Not Found 🚫 : Ressource introuvable. Exemple : une URL incorrecte.
  • 500 Internal Server Error 💥 : Erreur serveur. Exemple : un bug sur un site.
  • 202 Accepted 📥 : La demande est acceptée mais en cours de traitement.
Se Déplacer avec les Méthodes HTTP :
  • GET 📥 : Demander une ressource sans rien changer. Exemple : Aller sur `https://mon_site.com/photo.jpg`.
  • POST 📤 : Envoyer des données cachées dans le corps de la requête. Exemple : Remplir un formulaire.

Les formulaires HTML sont comme des ponts entre toi (l’utilisateur) et le serveur. Tu envoies des informations au serveur pour les traiter.

Action et Méthode :
  • Action 🎯 : Où vont les informations ?
  • Méthode 📬 : GET (visible dans l’URL) ou POST (caché dans le corps de la requête).
Exemple : Remplir et Envoyer un Formulaire

Dans cet exemple, avec la méthode POST, les informations sont envoyées de manière cachée au serveur.

Des imprévus peuvent survenir dans ton programme. Avec les exceptions, tu peux intercepter et gérer les erreurs sans arrêter tout le programme.

Exemple : Attraper une Erreur

try {
  let result = 10 / 0; // Erreur : division par zéro
} catch (err) {
  console.log('Erreur attrapée 🚨 :', err.message);
} finally {
  console.log('Fini ✔️');
}

try : On essaie d’exécuter un code.

catch : On attrape l’erreur si elle survient.

finally : Ce code s’exécute quoi qu'il arrive.

HTTPS : Sécuriser les Routes 🛡️

HTTPS est une version sécurisée d'HTTP. Les messages sont cryptés, personne ne peut les lire pendant le trajet.

Cookies 🍪 : Souvenirs de ton Exploration

Les cookies sont des morceaux d'informations stockés pour que les sites se souviennent de toi (comme des identifiants de connexion).

Les En-têtes (Headers) 📄

Les en-têtes HTTP ajoutent des détails à chaque requête ou réponse (exemple : le type de contenu envoyé).

Explication des Erreurs avec Try-Catch

Quand on programme, il est fréquent qu'une action échoue pour différentes raisons :
  • Variables manquantes : utiliser une variable qui n'a pas été définie.
  • Division par zéro : certaines opérations mathématiques ne sont pas possibles.
  • Problèmes de connexion : accéder à une ressource indisponible.
Les erreurs peuvent arrêter complètement ton programme si elles ne sont pas gérées correctement.

Avec try-catch, tu prévois les erreurs possibles et les gères pour éviter que ton programme ne s'arrête brutalement. Le bloc catch permet de capturer et réagir à ces erreurs sans arrêter l'exécution.

  • try : Tester du code qui pourrait échouer.
  • catch : Capturer l'erreur si elle survient et réagir.
  • finally (optionnel) : Exécuter ce bloc quoi qu'il arrive.

try {
  let result = 10 / 0;  // Erreur potentielle : division par zéro
  console.log(result);
} catch (error) {
  console.log("Une erreur s'est produite :", error.message);
} finally {
  console.log("Opération terminée.");
}

Le try-catch te permet de gérer les erreurs de manière propre et continue, sans arrêter ton programme. Cela t'offre plus de flexibilité pour réagir et prendre en compte les erreurs.
