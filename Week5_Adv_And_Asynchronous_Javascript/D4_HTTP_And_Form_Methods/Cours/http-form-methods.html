HTTP : La Route entre le Client et le Serveur

HTTP est un protocole stateless. Imaginez un facteur qui envoie des lettres au serveur sans garder de mémoire des précédentes. Chaque requête est indépendante des autres.

  • GET : Le facteur demande des informations au serveur, visibles dans l'URL.
  • POST : Le facteur envoie des informations sensibles au serveur dans le corps de la requête.

La requête HTTP contient trois parties principales :

  • Ligne de démarrage : Spécifie la méthode (GET, POST) et la version HTTP.
  • Headers : Donnent des informations supplémentaires sur la requête.
  • Body : Contient les données réelles envoyées au serveur.

La réponse HTTP contient trois éléments principaux :

  • Ligne de statut : Indique si la requête a réussi, par exemple "HTTP/1.1 200 OK".
  • Headers : Donnent des informations sur le format des données renvoyées.
  • Body : Contient les données réelles renvoyées par le serveur.

Quelques exemples de codes de statut :

  • 200 OK : La requête a réussi.
  • 404 Not Found : La ressource demandée n'a pas été trouvée.
  • 500 Internal Server Error : Une erreur s'est produite côté serveur.

Formulaires HTML

L'attribut action spécifie où les données du formulaire sont envoyées lorsqu'elles sont soumises.

Exemple :

<form action="https://example.com"></form>

L'attribut method définit comment les données sont envoyées. Il y a deux méthodes principales :

  • GET : Les données sont envoyées en tant que paramètres d'URL (visible dans l'URL).
  • POST : Les données sont envoyées dans le corps de la requête, non visibles dans l'URL.

Exemple :

<form method="POST" action="index.html"></form>

Avec la méthode GET, les données du formulaire sont ajoutées à l'URL sous forme de paires name=value.

Exemple :

<form method="GET" action="index.html">
                                                    <p>Name: <input type="text" name="name"></p>
                                                    <p>Message:<br><textarea name="message"></textarea></p>
                                                    <p><button type="submit">Send</button></p>
                                                  </form>

Lorsque vous soumettez ce formulaire, l'URL pourrait ressembler à :

index.html?name=John&message=Hello

Avec la méthode POST, les données sont envoyées dans le corps de la requête et non dans l'URL.

Exemple :

<form method="POST" action="index.html">
                                                    <p>Name: <input type="text" name="name"></p>
                                                    <p>Message:<br><textarea name="message"></textarea></p>
                                                    <p><button type="submit">Send</button></p>
                                                  </form>

Les données sont envoyées sous cette forme dans le corps de la requête :

Body "name=John&message=Hello"

Gestion des Exceptions en JavaScript

Analogie : Imagine que ton programme est une voiture. Quand quelque chose ne va pas (comme un pneu crevé), la voiture déclenche un signal d'alerte (une exception). Tu peux décider quoi faire avec cette alerte, par exemple, réparer le pneu et continuer de conduire. C'est exactement ce que fait une exception dans un programme : elle te signale un problème, et tu peux choisir comment le gérer.

try : Imagine que tu fais un test risqué dans ton programme, comme conduire sur une route avec des obstacles. Tu encadres ce test avec un bloc try pour voir si tout se passe bien.

catch : Si tu rencontres un obstacle (une exception), le bloc catch attrape ce problème et te permet de réagir (comme réparer le pneu).


                                          try {
                                            let x = 5;
                                            x = y + 1; // Erreur car "y" n'est pas défini
                                          } catch (err) {
                                            console.log("Erreur attrapée : ", err.message);
                                          }

Sortie :

Erreur attrapée : y is not defined

finally : Ce bloc contient du code qui est exécuté quoi qu'il arrive, même s'il y a une erreur ou non.


                                          try {
                                            console.log("Début du bloc try");
                                            hello; // Provoque une erreur
                                          } catch (err) {
                                            console.log("Erreur attrapée : ", err.message);
                                          } finally {
                                            console.log("Bloc finally exécuté");
                                          }

Sortie :


                                          Début du bloc try
                                          Erreur attrapée : hello is not defined
                                          Bloc finally exécuté

throw : Si tu détectes toi-même un problème, tu peux jeter l'alerte avec throw. C'est comme déclencher manuellement le signal d'alerte dans la voiture. 


                                          const diviser = (a, b) => {
                                            try {
                                              if (b === 0) {
                                                throw new Error("Impossible de diviser par zéro");
                                              }
                                              console.log("Résultat : ", a / b);
                                            } catch (err) {
                                              console.log("Erreur : ", err.message);
                                            }
                                          }
                                          
                                          diviser(10, 0); // Impossible de diviser par zéro

Voici quelques types d'erreurs intégrées dans JavaScript :

Erreur Description
ReferenceError Se produit lorsqu'une référence invalide est utilisée (comme une variable non définie).
SyntaxError Erreur de syntaxe (par exemple, code mal écrit ou manquant).
TypeError Se produit lorsqu'une valeur n'est pas du bon type.
RangeError Se produit lorsque la valeur est en dehors de l'intervalle autorisé.

Exemples Simples : Déstructuration et Passage par Valeur/Référence

Explication : La déstructuration permet d'extraire des valeurs d'un tableau et de les assigner à des variables distinctes.

Exemple :


            const arr = ['🥓', '🍕', '🍟'];
            const [bacon, pizza, fries] = arr;
            console.log(pizza); // 🍕

Dans cet exemple, les éléments du tableau arr sont assignés directement aux variables bacon, pizza et fries.

Explication : Quand tu passes une valeur primitive (comme un nombre), une copie est faite. Un changement sur une variable n'affecte pas l'autre.

Analogie : C'est comme copier une feuille de papier avec un nombre. Si tu changes ta feuille, l'autre copie ne change pas.

Exemple :


            let x = 10;      // Tu écris "10" sur une feuille de papier
            let y = x;       // Tu fais une copie de la feuille de papier pour quelqu'un d'autre
            x = 20;          // Tu changes ton propre papier en "20"
            console.log(y);  // L'autre personne voit toujours "10" sur sa copie

Ici, x a été modifié à 20, mais y reste à 10, car ils ne partagent pas la même feuille.

Explication : Quand tu passes un objet ou un tableau, tu donnes une référence à l'objet original. Changer l'objet affecte toutes les variables qui pointent vers lui.

Analogie : C'est comme partager le même plan de maison. Si quelqu'un modifie le plan, tout le monde voit le changement.

Exemple :


            let house = { color: 'blue' }; // Un plan de maison bleu
            let anotherHouse = house;      // Tu donnes le même plan à quelqu'un d'autre
            house.color = 'red';           // Tu changes la couleur sur le plan
            console.log(anotherHouse.color); // L'autre personne voit aussi "red"

Dans cet exemple, house et anotherHouse pointent vers le même objet. Changer la couleur dans house affecte aussi anotherHouse.

Passer par Valeur :

  • Comme copier un papier avec un nombre.
  • Chaque variable a sa propre copie, un changement dans une variable n'affecte pas l'autre.

Passer par Référence :

  • Comme partager le même plan de maison.
  • Toutes les variables pointent vers le même objet, un changement est visible par tous.

Exemples Simplifiés : HTTP, Formulaires et Exceptions

Explication : Imagine que ton ordinateur veut parler à un serveur. Pour ça, il envoie des messages !

GET 📥 : "Je veux quelque chose !" (par exemple, demander une image).

POST 📤 : "Je t'envoie quelque chose !" (comme un formulaire rempli).


// GET 📥 : "Je veux une image !"
fetch('https://site.com/image')
  .then(response => response.json())  // On reçoit l'image
  .then(data => console.log(data));   // On la montre

// POST 📤 : "Je t'envoie une lettre !"
fetch('https://site.com/form', {
  method: 'POST',
  body: JSON.stringify({ message: '💌' }) // On envoie une lettre
})
  .then(response => response.json())  // On reçoit la réponse
  .then(data => console.log(data));   // On la montre

Explication : Imagine que tu remplis une lettre (formulaire) et que tu veux l'envoyer.

GET 📬 : Tout est visible sur la carte postale (l'URL).

POST 📩 : Les infos sont cachées dans une enveloppe (pas visibles dans l'URL).

Explication : Parfois, il y a des problèmes dans ton programme, comme ajouter des pommes à des bananes 🍌.

try : On essaie de faire quelque chose.

catch : Si ça casse, on répare !


try {
  let x = y + 1;  // 💥 Oups, y n'existe pas !
} catch (err) {
  console.log('Erreur 🚨 :', err.message);  // On répare
} finally {
  console.log('Fini ✔️');  // On termine toujours ici
}
